 // 基础阈值\n\n// 速度越高，需要曲率越小才算急弯\nif (velocity > 10.0) {\n    curvature_threshold = 0.015;\n}\nelse if (velocity < 2.0) {\n    curvature_threshold = 0.05;\n}\n\nbool is_sharp_turn = std::abs(curvature) > curvature_threshold;\n```\n\n---\n\n### 问题 2: 路径宽度检查逻辑有缺陷\n\n**代码位置**: `DecidePathBounds()` 函数，约第 138-145 行\n\n```cpp\nelse if (is_sharp_turn && path_narrowest_width < 1.5) {\n    // S弯场景下，如果路径太窄（小于1.5米），也使用原始车道边界\n    AINFO << \"S-turn with narrow path width=\" << path_narrowest_width\n          << \", using original lane boundary for more space.\";\n    path_bound = original_path_bound;\n    blocking_obstacle_id = \"\";\n}\n```\n\n**问题分析**:\n\n| 参数 | 值 | 评估 |\n|-----|-----|------|\n| 路径宽度阈值 | 1.5 m | ⚠️ 过小 |\n| **车宽** | 2.9 m | ❌ **阈值 < 车宽** |\n| 最小安全间隙 | 0.7 m × 2 = 1.4 m | ❌ 几乎无容错空间 |\n| 实际需要宽度 | 2.9 + 1.4 = 4.3 m | ⚠️ 道路仅 4.1 m |\n\n**问题**:\n1. **阈值太低**: 1.5m < 2.9m(车宽)，无法通过\n2. **没有考虑车宽**: 应该是 `vehicle_width + 2 * safety_margin`\n3. **回退逻辑不安全**: 直接使用原始车道边界可能与障碍物碰撞\n\n**建议修复**:\n```cpp\ndouble min_required_width = config_.vehicle_width + 2 * config_.safety_margin;\n\nif (is_sharp_turn && path_narrowest_width < min_required_width) {\n    AINFO << \"S-turn with insufficient path width=\" << path_narrowest_width \n          << \" < required=\" << min_required_width;\n    \n    // 尝试多项策略而不是直接回退\n    if (!TryOptimizedAvoidancePath(...)) {\n        if (!TryOffsetPathBoundary(...)) {\n            AERROR << \"Cannot find feasible path in S-turn with obstacle\";\n            return false;
